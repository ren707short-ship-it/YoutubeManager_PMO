## システム概要

このシステムは、YouTube Shortsの量産運用を効率化するための管理ツールです。

### 主な機能
- **ユーザー管理**: オーナーと外注クリエイターの管理
- **アカウント管理**: 複数のYouTubeチャンネルの管理
- **動画タスク管理**: 6つのステータスで進捗管理
- **制作ポータル**: テンプレートとマニュアルの一元管理
- **参照チャンネル分析**: 成功チャンネルのパターン分析
- **フィードバック機能**: 納品物への確認・修正依頼
- **YouTube API連携**: 自動データ取得と分析

### ユーザーロール
1. **Owner（オーナー）**: 全機能にアクセス可能な管理者
2. **Creator（クリエイター）**: 担当タスクのみアクセス可能な外注者

## オーナー（管理者）向けマニュアル

### 1. ダッシュボードの使い方

ログイン後、最初に表示される画面です。

#### 表示内容
- **ステータス別動画数**: 各ステータスの動画タスク件数
- **フィードバック待ち一覧**: 確認が必要な動画の一覧
- **クリエイター別統計**: 各外注者のパフォーマンス
- **期限超過タスク**: 対応が必要な遅延タスク
- **期限が近いタスク**: 3日以内が期限のタスク

#### 操作方法
1. **動画タスクの確認**: タスクをクリックして詳細を表示
2. **クリエイター詳細**: 「詳細」ボタンでクリエイター情報を表示
3. **フィードバック完了**: 「完了にする」ボタンでFB完了

### 2. ユーザー管理

外注クリエイターの管理を行います。

#### クリエイターの追加手順
1. サイドバーから「ユーザー管理」をクリック
2. 「新規ユーザー作成」ボタンをクリック
3. 以下の情報を入力:
   - 名前（必須）
   - メールアドレス（必須）
   - パスワード（必須）
   - ロール: Creator を選択
   - 契約プラットフォーム（例: クラウドワークス）
   - 契約日
   - 契約書類URL（Google Driveなど）
   - ステータス: active（初期値）
   - 備考
4. 「作成」ボタンをクリック

#### クリエイター詳細の確認
1. ユーザー一覧で「詳細」ボタンをクリック
2. 表示される情報:
   - 基本情報（名前、メール、ロール）
   - 契約情報（プラットフォーム、契約日、契約書）
   - パフォーマンス統計（総タスク数、公開済み、FB待ち）
   - 提供マテリアル一覧
3. 「マテリアル追加」で素材を追加可能

#### クリエイターへのマテリアル提供
1. クリエイター詳細画面で「マテリアル追加」をクリック
2. 以下を入力:
   - 名前（例: FANZAテンプレート）
   - 種類: 動画/画像/テンプレート/その他
   - URL（Google DriveやDropboxのリンク）
3. 「追加」をクリック
4. クリエイターはログイン後、自分用のマテリアルを確認可能

### 3. アカウント管理

YouTubeチャンネルの管理を行います。

#### アカウントの追加手順
1. サイドバーから「アカウント」をクリック
2. 「新規アカウント作成」ボタンをクリック
3. 以下の情報を入力:
   - アカウント名（必須）
   - ジャンル（例: FANZA作品紹介）
   - URL（YouTubeチャンネルURL）
   - **YouTube チャンネルID**（API連携用、重要！）
   - 担当クリエイター（割り当てる外注者）
4. 「作成」ボタンをクリック

#### YouTube チャンネルIDの取得方法
1. YouTubeで自分のチャンネルを開く
2. URLを確認: `https://www.youtube.com/channel/【これがチャンネルID】`
3. または、チャンネルページの「概要」→「チャンネルIDをコピー」

#### アカウント詳細の使い方
1. アカウント一覧で「👁」アイコンをクリック
2. タブで切り替え:
   - **動画タスク**: ステータス別のカンバン表示
   - **マテリアル**: チャンネル専用の素材管理
   - **参照チャンネル**: 参考にする成功チャンネル

### 4. 動画タスク管理

#### 動画タスクの作成手順
1. サイドバーから「動画」をクリック
2. 「新規動画タスク作成」ボタンをクリック
3. **「テンプレから作成」を選択**（推奨）
   - テンプレートを選択すると台本が自動入力されます
4. 以下の情報を入力:
   - アカウント（必須）
   - 担当クリエイター
   - タイトル（必須）
   - テンプレート（**v2の新機能！**）
   - ステータス（初期値: アイデア）
   - 期限
   - アフィリエイトリンク
   - **フィードバック必須**: チェックで確認フローを有効化
   - **フィードバック期限**: FB完了目標日
   - 台本（テンプレから自動入力）
   - 台本（英訳）
   - アセットリンク
5. 「作成」ボタンをクリック

#### ステータスの種類と意味
1. **アイデア**: ネタ段階、まだ制作開始前
2. **台本中**: 台本を作成中
3. **編集中**: 動画を編集中
4. **確認待ち**: オーナーの確認待ち（FB段階）
5. **予約済み**: YouTube予約投稿済み
6. **投稿済み**: 公開完了

#### フィードバック機能の使い方
1. 動画詳細画面を開く
2. **フィードバック待ち**の表示を確認
   - 紫色の表示: FB待ち
   - 緑色の表示: FB完了
3. 動画を確認後、「完了にする」ボタンをクリック
4. またはコメント機能で修正依頼を送信

### 5. 制作ポータルの管理

#### テンプレートの作成
1. サイドバーから「制作ポータル」をクリック
2. 「テンプレート追加」ボタンをクリック
3. 以下を入力:
   - テンプレート名（例: FANZA作品紹介_3カット）
   - カテゴリ（例: fanza_intro）
   - 説明
   - **CapCutプロジェクトURL**（重要！）
   - 日本語台本テンプレート（プレースホルダ使用可）
   - 英訳台本テンプレート
   - 注意事項
4. 「作成」ボタンをクリック

#### プレースホルダの使い方
台本テンプレートでは以下の形式でプレースホルダを使用できます:
```
{{character_name}}の{{situation}}が話題！
詳細は概要欄から👇
```

動画作成時に`{{}}` 部分を実際の内容に置き換えます。

#### マニュアルの作成・編集
1. 「マニュアル追加」ボタンをクリック
2. 以下を入力:
   - タイトル（例: FANZA作品紹介_運用ガイド）
   - カテゴリ（テンプレートと同じカテゴリ推奨）
   - 内容（**Markdown形式**で記述）
3. 「作成」ボタンをクリック

#### Markdown形式の例
```markdown
# マニュアルタイトル

## 基本ルール
- ルール1
- ルール2

## 注意事項
**太字** や *斜体* を使用可能

### サブセクション
1. 手順1
2. 手順2

> 引用文

\`\`\`
コードブロック
\`\`\`
```

### 6. 参照チャンネル管理

成功している他のチャンネルを分析します。

#### 参照チャンネルの追加
1. アカウント詳細画面の「参照チャンネル」タブを開く
2. 「参照チャンネル追加」ボタンをクリック
3. 以下を入力:
   - チャンネル名
   - **YouTube チャンネルID**（API連携用）
   - ジャンル（分類用）
   - メモ（なぜ参考にするか）
4. 「追加」ボタンをクリック

#### 参考動画の追加
1. 参照チャンネル一覧で「参考動画を見る」をクリック
2. 「参考動画追加」ボタンをクリック
3. 以下を入力:
   - 動画タイトル
   - YouTube 動画ID（URLの `v=` の後の部分）
   - 分析メモ（なぜこの動画がうまくいったか）
4. 「追加」ボタンをクリック

### 7. YouTube API連携（v2新機能）

#### API設定の手順
1. サイドバーから「設定」をクリック
2. 「YouTube API設定」セクションを開く
3. **YouTube Data API v3のAPIキーを取得**:
   - Google Cloud Console にアクセス
   - プロジェクトを作成
   - YouTube Data API v3を有効化
   - APIキーを作成
4. 取得したAPIキーを入力
5. 「保存」ボタンをクリック

#### 自アカウントの動画を自動同期
1. アカウント管理画面を開く
2. 各アカウントに**YouTube チャンネルID**を設定
3. 「動画を同期」ボタンをクリック
4. 自動的に:
   - 最新50件の動画を取得
   - タイトル、再生数、いいね数を更新
   - 新しい動画は自動的にタスクとして追加

#### 参照チャンネルの動画を自動取得
1. 参照チャンネル詳細画面を開く
2. 「動画を同期」ボタンをクリック
3. 人気動画50件を自動取得
4. 再生数順に表示され、分析に活用可能

### 8. アイデア管理

3. 以下を入力:

### 9. アフィリエイトリンク管理

#### アフィリエイトの追加

2. 「新規アフィリエイトリンク作成」ボタンをクリック

## クリエイター（外注）向けマニュアル

### 1. ログインから作業開始まで

#### ログイン手順
1. 指定されたURLにアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック

#### ダッシュボードの見方
- **自分のタスク**: 担当している動画タスクの一覧
- **期限超過**: 赤色で表示される遅延タスク
- **期限が近い**: 黄色で表示される注意タスク

### 2. 動画タスクの作業フロー

#### ステップ1: タスクを開く
1. ダッシュボードまたは「動画」ページから担当タスクを選択
2. タスク詳細画面が開く

#### ステップ2: マニュアルを確認
1. 動画詳細画面で「マニュアルを開く」リンクをクリック
2. ジャンル別のマニュアルを熟読
3. **重要**: コンプラや禁止事項を必ず確認

#### ステップ3: テンプレートをダウンロード
1. 動画詳細画面で紐づいているテンプレートを確認
2. 「CapCutテンプレDL」ボタンをクリック
3. CapCutプロジェクトファイル（.ccproj）をダウンロード

#### ステップ4: 台本を確認
1. 動画詳細画面の「台本」セクションを確認
2. `{{}}` プレースホルダがある場合は実際の内容に置き換え
3. 日本語台本と英訳台本の両方を確認

#### ステップ5: 素材を確認
1. 「アセットリンク」セクションを確認
2. Google DriveやDropboxから必要な素材をダウンロード
3. マイページの「提供マテリアル」も確認

#### ステップ6: 編集作業
1. CapCutでテンプレートを開く
2. 台本に沿ってテキストを差し替え
3. 素材（画像・動画）を差し替え
4. マニュアルの注意事項を守る

#### ステップ7: YouTube投稿
1. 動画を書き出し
2. YouTubeに投稿
3. **投稿したURLをコピー**

#### ステップ8: ステータス更新
1. 動画詳細画面に戻る
2. 「編集」ボタンをクリック
3. 以下を更新:
   - ステータス: `確認待ち` に変更
   - YouTube URL: コピーしたURLを貼り付け
4. 必要に応じてコメントで報告
5. 「更新」ボタンをクリック

### 3. フィードバックへの対応

#### FB待ち表示の確認
- 動画詳細画面に紫色の「フィードバック待ち」表示
- 期限も表示されるので注意

#### FB内容の確認
1. コメント欄を確認
2. オーナーからの指示を読む
3. 修正箇所をメモ

#### 修正作業
1. CapCutで動画を修正
2. 再度YouTubeに投稿（または既存動画を差し替え）
3. ステータスを `確認待ち` に戻す
4. コメントで「修正完了しました」と報告

### 6. よくある質問（FAQ）

#### Q1: パスワードを忘れました
A: オーナーに連絡してパスワードをリセットしてもらってください。

#### Q2: 担当タスクが表示されません
A: オーナーがまだタスクを割り当てていない可能性があります。連絡してください。

#### Q3: テンプレートがダウンロードできません
A: URLが正しいか確認してください。Google Driveの場合、共有設定を確認してください。

#### Q4: アフィリエイトリンクが見えません
A: クリエイターには実際のURLは表示されません。内部名称のみ表示されます。

#### Q5: マニュアルが見つかりません
A: 制作ポータルまたは動画詳細画面から該当ジャンルのマニュアルにアクセスしてください。

## YouTube API連携

### API取得の前提知識

#### YouTube Data API v3とは
- Googleが提供する無料API
- 動画情報、チャンネル情報、統計情報を取得可能
- 1日の無料枠: 10,000ユニット

#### APIキーの取得手順（詳細）

##### ステップ1: Google Cloud Console にアクセス
1. https://console.cloud.google.com/ にアクセス
2. Googleアカウントでログイン

##### ステップ2: プロジェクトを作成
1. 上部の「プロジェクトを選択」をクリック
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を入力（例: youtube-shorts-manager）
4. 「作成」ボタンをクリック

##### ステップ3: APIを有効化
1. 左メニューから「APIとサービス」→「ライブラリ」を選択
2. 検索バーで「YouTube Data API v3」を検索
3. 「YouTube Data API v3」をクリック
4. 「有効にする」ボタンをクリック

##### ステップ4: APIキーを作成
1. 左メニューから「APIとサービス」→「認証情報」を選択
2. 上部の「認証情報を作成」をクリック
3. 「APIキー」を選択
4. APIキーが表示されたらコピー
5. **重要**: 「キーを制限」をクリックして制限を設定推奨

##### ステップ5: APIキーの制限（推奨）
1. 「アプリケーションの制限」: なし または IPアドレス
2. 「API の制限」: 「キーを制限」を選択
3. 「YouTube Data API v3」のみにチェック
4. 「保存」ボタンをクリック

### 自アカウント同期の詳細

#### 同期の仕組み
1. 設定されたYouTube チャンネルIDから動画一覧を取得
2. 最新50件の動画情報を取得
3. 既存動画は再生数・いいね数を更新
4. 新規動画は自動的にタスクとして追加

#### 同期の実行
1. アカウント管理画面を開く
2. 対象アカウントの「動画を同期」ボタンをクリック
3. 数秒〜数十秒で完了
4. 結果が表示される（同期数、新規作成数）

#### 同期後の確認
1. 動画一覧画面を開く
2. 新規追加された動画を確認
3. ステータスは自動的に `published` になる
4. 再生数・いいね数が反映されている

### 参照チャンネル同期の詳細

#### 同期の仕組み
1. 参照チャンネルのYouTube チャンネルIDから人気動画を取得
2. 再生数順に最大50件を取得
3. タイトル、説明、再生数、いいね数、長さを保存
4. 既存動画は統計を更新

#### 同期の実行
1. アカウント詳細の「参照チャンネル」タブを開く
2. 対象チャンネルの「動画を同期」ボタンをクリック
3. 結果が表示される

#### 分析の活用
1. 人気動画一覧を再生数順に確認
2. タイトルの傾向を分析
3. 動画の長さを確認（15秒が多い？30秒？）
4. 説明文のパターンを研究
5. 自分のネタやテンプレに反映

## GitHub管理とデプロイ

### GitHub連携の基本

#### 推奨: GitHubでコード管理

このシステムは以下の方法で管理・共有できます:

1. **GitHub Repository**: コードをGitHubで管理（推奨）
2. **直接URL共有**: デプロイ後のURLを共有
3. **バックアップファイル**: tar.gzファイルで共有

### GitHub Repository の作成

#### ステップ1: GitHubアカウント作成
1. https://github.com/ にアクセス
2. 「Sign up」でアカウント作成
3. メール確認を完了

#### ステップ2: 新しいRepositoryを作成
1. GitHubにログイン
2. 右上の「+」アイコン→「New repository」
3. Repository名を入力（例: youtube-shorts-manager）
4. Private を選択（推奨）
5. 「Create repository」をクリック

#### ステップ3: ローカルコードをPush

##### 開発環境（Sandbox）から:
```bash
cd /home/user/webapp

# GitHub環境のセットアップ（初回のみ）
# このツールが自動的にgitとghを設定します
# 設定後、通常のgit/ghコマンドが使えます

# リモートを追加
git remote add origin https://github.com/your-username/youtube-shorts-manager.git

# ブランチ名を main に変更（まだの場合）
git branch -M main

# Push
git push -u origin main
```

#### ステップ4: チームメンバーを招待（必要な場合）
1. Repository ページの「Settings」タブ
2. 左メニューの「Collaborators」
3. 「Add people」でメールアドレスを入力
4. 権限を選択（Write推奨）

### Cloudflare Pages へのデプロイ

#### 前提条件
- Cloudflareアカウント（無料）
- GitHubにコードをpush済み

#### ステップ1: Cloudflare Pages プロジェクト作成

##### 方法A: GitHub連携（推奨）
1. Cloudflare Dashboard にログイン
2. 「Workers & Pages」→「Create application」
3. 「Pages」タブ→「Connect to Git」
4. GitHubアカウントを連携
5. Repositoryを選択
6. Build設定:
   - **Build command**: `npm run build`
   - **Build output directory**: `dist`
7. 「Save and Deploy」をクリック

##### 方法B: Wrangler CLI（高度）
```bash
cd /home/user/webapp

# Cloudflare APIキーを設定（初回のみ）
# 設定画面からAPIキーを取得しておく

# ビルド
npm run build

# デプロイ
npx wrangler pages deploy dist --project-name youtube-shorts-manager
```

#### ステップ2: D1データベースの設定

##### 本番D1データベースを作成:
```bash
# 本番データベースを作成
npx wrangler d1 create youtube-shorts-manager-production

# 出力されたdatabase_idをコピー
# wrangler.jsonc に記載
```

##### wrangler.jsonc を更新:
```jsonc
{
  "name": "youtube-shorts-manager",
  "compatibility_date": "2024-01-01",
  "d1_databases": [
    {
      "binding": "DB",
      "database_name": "youtube-shorts-manager-production",
      "database_id": "YOUR_DATABASE_ID_HERE"  // ここに貼り付け
    }
  ]
}
```

##### マイグレーションを適用:
```bash
# すべてのマイグレーションを本番に適用
npx wrangler d1 execute youtube-shorts-manager-production --file=./migrations/0001_initial_schema.sql
npx wrangler d1 execute youtube-shorts-manager-production --file=./migrations/0002_add_advanced_features.sql
npx wrangler d1 execute youtube-shorts-manager-production --file=./migrations/0003_v2_production_portal_clean.sql

# サンプルデータを投入（任意）
npx wrangler d1 execute youtube-shorts-manager-production --file=./seed.sql
npx wrangler d1 execute youtube-shorts-manager-production --file=./seed_advanced.sql
npx wrangler d1 execute youtube-shorts-manager-production --file=./seed_v2.sql
```

#### ステップ3: 環境変数の設定（必要な場合）
1. Cloudflare Dashboard の Pages プロジェクト
2. 「Settings」→「Environment variables」
3. 必要な変数を追加

#### ステップ4: カスタムドメイン設定（任意）
1. 「Custom domains」タブ
2. 「Set up a custom domain」
3. 自分のドメインを入力
4. DNSレコードを設定

## トラブルシューティング

### ログインできない

#### 原因1: パスワードが間違っている
- **解決**: 正しいパスワードを入力
- **オーナーの場合**: データベースをリセットして初期パスワードに戻す
- **クリエイターの場合**: オーナーに連絡してパスワードをリセットしてもらう

#### 原因2: セッションが切れている
- **解決**: ページをリロードしてもう一度ログイン

### 動画タスクが表示されない

#### 原因1: 担当が割り当てられていない（クリエイター）
- **解決**: オーナーに連絡して担当を割り当ててもらう

#### 原因2: フィルターがかかっている
- **解決**: ステータスフィルターを「すべて」に変更

### YouTube API が動作しない

#### 原因1: APIキーが設定されていない
- **解決**: 設定画面でAPIキーを入力

#### 原因2: APIキーの制限設定が厳しい
- **解決**: Google Cloud Consoleで制限を緩和

#### 原因3: YouTube チャンネルIDが間違っている
- **解決**: 正しいチャンネルIDを入力（`UC`で始まる文字列）

#### 原因4: API利用枠を超えた
- **解決**: 翌日まで待つ、または有料枠にアップグレード

### テンプレートがダウンロードできない

#### 原因1: URLが間違っている
- **解決**: 正しいURLを入力

#### 原因2: Google Driveの共有設定が間違っている
- **解決**: 「リンクを知っている全員」に設定

### マニュアルが表示されない

#### 原因1: カテゴリが一致していない
- **解決**: テンプレートとマニュアルのカテゴリを同じにする

#### 原因2: マニュアルが作成されていない
- **解決**: オーナーがマニュアルを作成する

### サーバーエラー（500エラー）

#### 原因: データベース接続エラー
- **解決1**: サーバーを再起動
  ```bash
  pm2 restart webapp
  ```
- **解決2**: データベースをリセット
  ```bash
  npm run db:reset
  ```

### ポート3000が使用中

#### 解決方法:
```bash
# ポートをクリーンアップ
npm run clean-port

# または直接killする
fuser -k 3000/tcp
```
